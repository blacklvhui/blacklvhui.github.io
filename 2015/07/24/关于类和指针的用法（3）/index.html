关于类和指针的用法（3）

任何事物都可以看作是对象。

关于头文件的引用。
cpp文件引用头文件：用到哪些头文件就进行引用，因为不会造成下一层的嵌套，因为不会对cpp文件进行引用。
.h文件引用.h文件时，不应用到什么就引用什么，因为会造成多次的嵌套，到时候会造成一些不必要的麻烦。

关于返回值
例如get&set中的用法：

int identity() const { return identity_; }
void set_identity(int identity) { identity_ = identity; }

C++关于变量命名的风格

变量名一律小写, 单词之间用下划线连接. 类的成员变量以下划线结尾, 如:

my_exciting_local_variable
my_exciting_member_variable_

写程序时，尽量调理清晰，任何事物都可以看作是对象，可以采用类使调理、逻辑清晰。

有些时候写判断、循环的时候写while（a == 1） 不如写while(a != 0)，一定要考虑清楚。

构造函数和析构函数

#pragma mark - Init
public:
    User();
    ~User();

初始化方法：

User::User():
identity_(0),
name_("no na1me"),
price_(0.0)
{
}
